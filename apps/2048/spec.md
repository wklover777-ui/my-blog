# 2048 퍼즐 게임 스펙

## 1. 게임 개요

4×4 그리드 위에서 방향키(또는 터치 스와이프)로 타일을 밀어 같은 숫자끼리 합치는 퍼즐 게임.
타일이 합쳐지면 숫자가 2배가 되며, 2048 타일을 만들면 승리한다.
이동할 때마다 빈 칸에 새 타일(2 또는 4)이 랜덤으로 생성된다.
이동 가능한 칸이 없으면 게임 오버.

기술 스택: 순수 HTML / CSS / JavaScript — 빌드 도구 없음, CDN 허용.
블로그 루트(`my-blog/`)의 기존 파일은 일절 수정하지 않는다.

## 2. 파일 구조

```
my-blog/
└── apps/
    └── 2048/
        ├── index.html      # 게임 진입점
        ├── spec.md         # 이 파일
        ├── css/
        │   └── style.css   # 전체 스타일 (그리드, 타일 색상, 애니메이션, 반응형)
        └── js/
            ├── game.js     # 게임 로직 (보드 상태, 이동, 병합, 점수)
            ├── render.js   # DOM 렌더링 (보드 그리기, 타일 업데이트)
            └── input.js    # 입력 처리 (키보드, 터치 스와이프)
```

## 3. 핵심 기능 목록

### 게임 로직 (game.js)
- 4×4 이중 배열로 보드 상태 관리
- 상/하/좌/우 이동 함수: 타일을 끝까지 밀고 인접한 같은 숫자 병합
- 한 번의 이동에서 같은 타일은 최대 1회만 병합 (연쇄 병합 금지)
- 이동 후 빈 셀에 랜덤 타일(2: 90%, 4: 10%) 생성
- 점수 계산: 병합 시 합산된 숫자만큼 점수 누적
- 최고 점수(best score) localStorage에 저장 및 로드
- 승리 조건: 2048 타일 생성 시 승리 메시지 표시 (계속 플레이 가능)
- 게임 오버 조건: 빈 셀 없음 + 인접한 같은 숫자 없음
- 새 게임 시작: 보드 초기화 후 타일 2개 생성

### 렌더링 (render.js)
- 보드 상태 배열 → DOM 업데이트
- 타일 숫자별 배경색 CSS 클래스 적용 (tile-2, tile-4, … tile-2048, tile-super)
- 타일 등장 애니메이션 (scale 0 → 1)
- 타일 병합 애니메이션 (pop 효과)
- 현재 점수 / 최고 점수 실시간 업데이트
- 게임 오버 / 승리 오버레이 표시 및 숨기기

### 입력 처리 (input.js)
- 키보드: ArrowUp, ArrowDown, ArrowLeft, ArrowRight
- 터치: touchstart / touchend 이벤트로 스와이프 방향 감지
  - 최소 스와이프 거리 30px (오작동 방지)
  - 수직/수평 중 더 큰 방향 우선
- 이동 중(애니메이션 진행 중) 입력 무시 (디바운스)

## 4. UI 구성

### 헤더
- 왼쪽: 게임 타이틀 "2048"
- 오른쪽: 점수판 2개 (SCORE / BEST) + 새 게임 버튼

### 게임 그리드
- 4×4 CSS Grid, 셀 간격(gap) 10px
- 셀 크기: 데스크톱 100px × 100px / 모바일 70px × 70px (vw 기반)
- 빈 셀: 배경색(#cdc1b4)으로 항상 표시 (16개 고정)
- 타일: border-radius 6px

### 오버레이
- 게임 오버: "Game Over!" 메시지 + 다시 시작 버튼
- 승리: "You Win!" 메시지 + "계속 플레이" / "새 게임" 버튼

### 반응형
- 브레이크포인트 480px 이하: 그리드·타일 크기 vw 기반으로 축소
- 터치 시 페이지 스크롤 방지
