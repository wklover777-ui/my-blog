클로드코드와 함께 블로그를 처음부터 만들어봤다. 코드를 직접 치지 않아도 대화하면서 완성되는 과정이 신기했는데, 오히려 그 덕분에 각 파일이 무슨 일을 하는지 더 선명하게 보였다.

## HTML — 뼈대

HTML은 페이지에 무엇이 있는지를 정의한다. 이 블로그에서는 두 개의 HTML 파일이 전부다.

- `index.html` — 포스트 목록을 보여주는 페이지
- `post.html` — 개별 글을 렌더링하는 페이지

HTML 파일 안에는 헤더, 본문, 푸터 같은 구조만 있고, 실제 포스트 내용은 JavaScript가 나중에 채워 넣는다. 뼈대만 세우고 살은 나중에 붙이는 방식이다.

## CSS — 옷

CSS는 HTML이 어떻게 보일지를 결정한다. 이 블로그는 CSS 파일을 세 개로 나눴다.

- `base.css` — 색상, 폰트 같은 기본값. 다크모드는 여기서 CSS 변수(`--color-bg`, `--color-text`)를 바꾸는 것만으로 전체 테마가 바뀐다.
- `layout.css` — 헤더 위치, 카드 그리드, 반응형 레이아웃. 화면 너비에 따라 카드가 2열에서 1열로 바뀌는 것도 CSS 한 줄(`@media`)로 처리한다.
- `post.css` — 마크다운이 변환된 HTML의 스타일. `h2`에 구분선을 긋거나, 코드 블록에 배경색을 입히는 것들이다.

CSS를 파일로 분리하니까 "이 스타일은 어디 있더라"를 금방 찾을 수 있었다.

## JavaScript — 움직임

JavaScript는 페이지를 살아 움직이게 한다. 이 블로그에서 JS가 하는 일은 크게 세 가지다.

**1. 포스트 목록 불러오기**  
`posts/index.json`을 서버에서 가져와서(`fetch`) 카드 HTML을 만들어 화면에 붙인다. HTML에는 빈 `<div id="post-grid">`만 있고, JS가 내용을 채운다.

**2. 마크다운 → HTML 변환**  
포스트 페이지에서 `.md` 파일을 가져온 다음, `marked.js` 라이브러리로 HTML로 바꿔 렌더링한다. 파일 맨 위의 `---` 블록(frontmatter)은 직접 파싱해서 제목, 날짜, 태그를 추출한다.

**3. 다크모드 토글**  
버튼을 누르면 `<html>` 태그에 `data-theme="dark"` 속성을 붙이거나 뗀다. CSS는 그 속성을 보고 변수를 바꾼다. 선택한 테마는 `localStorage`에 저장해서 새로고침해도 기억한다.

## 정리

| 역할 | 담당 | 한 줄 요약 |
|------|------|------------|
| 구조 | HTML | 무엇이 있는가 |
| 표현 | CSS | 어떻게 보이는가 |
| 동작 | JavaScript | 어떻게 움직이는가 |

세 가지가 각자 할 일만 하고 서로 간섭하지 않을수록 코드가 읽기 쉬워진다는 걸 이번에 실감했다. 인라인 스타일 금지, CSS 파일 분리, JS는 DOM 조작만 — 이런 원칙들이 왜 있는지 이제 조금 알 것 같다.
